The Software Evaluation

"Here is what an expert would say about this product."

A $20K evaluation. Seven dimensions. Honest scoring.
7
/10
Architecture
Clean separation — React + Vite + Tailwind, Supabase backend, Vercel serverless APIs. worker_notes as single source of walk state. Four-filter decision framework keeps scope honest.
Tower Control will test whether architecture holds with two different interfaces sharing one data layer. Need clean component boundaries.
7
/10
Walker App UX
WWRS baked in. Dog cards show name, address, door code without tap. Owl notes animate on name. Drawer opens to action centre. One-hand design visible.
Fix 4 bugs—especially interlock compact (BUG 4) which breaks multi-walker use. Then device-test end-to-end.
3
/10
Tower Control UX
Only Mini Gen Review Panel is live. Full Tower is in Figma—10 components and 8 screens designed but not built. Gen uses Google Sheets V3.9 as fallback.
Build Today view. Single highest-leverage screen. Replaces Gen's morning routine of checking 4 different sources.
8
/10
Data Model
worker_notes as single source of walk state is strong. dog_ids as text[] (names, not UUIDs) works because names are unique per sector. acuity_name_map at 74 entries resolves 100% of bookings.
Consider indexing walker_notes by walk_date for performance. expected_schedule (84 rows) may need more structure as operation grows.
6
/10
AI Integration (Mini Gen + Beast)
Mini Gen is standout—real production agent doing mechanical admin with 100% accuracy. Writes to staging, requires human approval before live. "Stop and wait" philosophy is exactly right. Beast is concept-only.
Build Beast with same discipline—confirm-before-act, never direct production access. Confirm block UI is designed; needs implementation.
4
/10
Completeness
Walker app core works. Mini Gen + promote works. But Tower is mostly in Figma, billing is vision-only, website is on Wix, Beast is concept. ~35-40% of surface area is built.
Phase 1 (Tower Today) and Phase 2 (Close Loop) are highest-leverage builds. Completing these brings completeness to ~65%.
5
/10
Production Readiness
95 dogs and 7 walkers use daily. Mini Gen runs successfully. But 4 bugs including one that breaks interlock (critical for multi-walker). Undo has state sync issue. Not-walking untested. Gen still uses Google Sheets.
Fix 4 bugs, device-test every flow, run regression. Production readiness jumps to 7+ when bugs resolved and Gen moves to Tower.
What Is Impressive & Must Be Protected
The design philosophy. WWRS, the four filters, "no caps without data," "agents stop and wait"—this is a solo founder who thinks like a senior architect. The discipline of worker_notes as single source of truth, dog names as text[], and Mini Gen's staging-table-first approach shows someone building for durability, not speed. Protect the decision-making framework. It's worth more than any single feature.
Single Biggest Risk
Tower Control. Gen still uses Google Sheets. Every day Tower doesn't exist, the operational bottleneck stays in Gen's head and Rod's text messages. Mini Gen writes perfect drafts that nobody can efficiently review because the review surface is a single panel. The loop is technically connected but practically still has a human-shaped gap.
Recommended Next Action
Fix the 4 walker app bugs (1-2 sessions), then build Tower Today view—the morning briefing screen that replaces Gen's Google Sheets routine. This single screen unblocks the entire operation.
